This does two things\+:
\begin{DoxyEnumerate}
\item Sets up the seed for random generator based on the arguments (excluding the executable name). That is, {\ttfamily ./a.out -\/n 10} and {\ttfamily ./b.out -\/n 10} will use the same seed.
\item Parses the opts (given in the arguments).
\end{DoxyEnumerate}\hypertarget{group__opts_autotoc_md14}{}\doxysubsubsubsection{\texorpdfstring{Opts}{Opts}}\label{group__opts_autotoc_md14}
Opts are a list of either named or positional options.

Named options is given in one of the following formats\+:
\begin{DoxyEnumerate}
\item {\ttfamily -\/keyname=value} or {\ttfamily -\/-\/keyname=value} (ex. {\ttfamily -\/n=10}, {\ttfamily -\/-\/test-\/count=20})
\item {\ttfamily -\/keyname value} or {\ttfamily -\/-\/keyname value} (ex. {\ttfamily -\/n 10}, {\ttfamily -\/-\/test-\/count 20})
\end{DoxyEnumerate}

Positional options are numbered from 0 sequentially.

For example, if you compile your code to {\ttfamily a.\+out}, calling {\ttfamily ./a.out -\/n 10 str -\/-\/q=5 -\/1.\+1e2} results in the following opts\+:
\begin{DoxyItemize}
\item Named
\begin{DoxyItemize}
\item {\ttfamily n}\+: {\ttfamily 10}
\item {\ttfamily q}\+: {\ttfamily 5}
\end{DoxyItemize}
\item Positional
\begin{DoxyItemize}
\item {\ttfamily 0}\+: {\ttfamily str}
\item {\ttfamily 1}\+: {\ttfamily -\/110}
\end{DoxyItemize}
\end{DoxyItemize}\hypertarget{group__opts_autotoc_md15}{}\doxysubsubsubsubsection{\texorpdfstring{Consuming opts}{Consuming opts}}\label{group__opts_autotoc_md15}
